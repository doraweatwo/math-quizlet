# Math Quizlet 모바일 앱 - 인터페이스 설계

## 설계 원칙
- **플랫폼**: iOS/Android 모두 지원 (Expo)
- **방향**: 세로 우선, 가로 모드도 완전 지원
- **기기**: 폰(5~6인치) 및 태블릿(7~10인치) 모두 최적화
- **스타일**: iOS HIG 준수, 깔끔하고 직관적인 UI

---

## 화면 구조

### 1. 홈 화면 (Home)
**목적**: 문제집 관리 및 학습 시작

**콘텐츠**:
- 상단: "My Problem Sets" 제목
- 중앙: 문제집 카드 리스트 (스크롤 가능)
  - 각 카드: 문제집 제목, 문제 수, 마지막 학습 날짜
- 하단: "+" 버튼 (새 문제집 추가)

**기능**:
- 문제집 탭: 학습 시작
- 문제집 길게 누르기: 삭제 옵션

---

### 2. PDF 업로드 화면 (PDFUpload)
**목적**: PDF 파일 선택 및 업로드

**콘텐츠**:
- 상단: "Import Problem Set" 제목
- 중앙: 큰 업로드 영역 (드래그 & 드롭 또는 탭)
- 하단: "Cancel" / "Next" 버튼

**기능**:
- 파일 선택 후 자동으로 다음 화면으로 이동

---

### 3. 패턴 설정 화면 (PatternSetup)
**목적**: 문제/해설 번호 패턴 지정

**콘텐츠**:
- 상단: "Set Pattern" 제목 + 진행률 (1/3)
- 중앙: PDF 미리보기 (페이지 네비게이션 포함)
  - 페이지 선택 슬라이더
  - 확대/축소 버튼
- 하단: 
  - "문제 번호 영역 지정" 버튼
  - "해설 번호 영역 지정" 버튼
  - "Preview" / "Next" 버튼

**기능**:
- 드래그로 영역 선택
- 선택 영역 시각적 하이라이트
- 패턴 미리보기 (1~2페이지 적용 결과)

---

### 4. 카드 추출 미리보기 화면 (CardPreview)
**목적**: 추출된 카드 확인 및 조정

**콘텐츠**:
- 상단: "Review Cards" 제목 + 진행률 (2/3)
- 중앙: 카드 리스트 (스크롤 가능)
  - 각 카드: 문제 이미지 + 해설 이미지 (좌우 배치)
- 하단: "Back" / "Confirm" 버튼

**기능**:
- 카드 스와이프로 삭제 (오류 제거)
- 카드 탭으로 상세 보기

---

### 5. 학습 화면 (StudySession)
**목적**: 문제 풀이 및 학습

**콘텐츠**:
- 상단: 문제집 제목 + 진행률 (예: 5/20)
- 중앙: 문제 이미지 (큰 영역)
- 하단:
  - 타이머 (경과 시간)
  - "정답" / "오답" 버튼
  - 난이도 별점 (1~5)

**기능**:
- 문제 자동 타이머 시작
- 정답/오답 선택 시 자동으로 다음 문제로 이동
- 난이도 별점 선택 가능
- 스와이프로 이전/다음 문제 이동

---

### 6. 해설 화면 (SolutionView)
**목적**: 문제 해설 확인

**콘텐츠**:
- 상단: "Solution" 제목
- 중앙: 해설 이미지 (전체 화면)
- 하단: "Back" 버튼

**기능**:
- 확대/축소 (핀치)
- 스크롤 (긴 해설)

---

### 7. 결과 화면 (SessionResult)
**목적**: 학습 세션 결과 요약

**콘텐츠**:
- 상단: "Session Complete" 제목
- 중앙:
  - 정답률 (%)
  - 총 풀이 시간
  - 평균 문제당 시간
  - 오답 문제 수
- 하단: "Home" / "Review Mistakes" 버튼

**기능**:
- 오답 문제만 다시 풀기 옵션

## 반응형 레이아웃 전략

### 폰 (< 600px)
- 단일 열 레이아웃
- 카드 미리보기: 문제/해설 위아래 배치
- 버튼: 풀 너비

### 태블릿 (≥ 600px)
- 2단 레이아웃 (가능한 경우)
- 카드 미리보기: 문제/해설 좌우 배치
- 버튼: 적절한 너비

### 가로 모드
- 높이 제약 고려
- 스크롤 영역 최소화
- 버튼 배치 조정

## 주요 UX 고려사항

1. **한 번에 완료**: 모든 흐름이 명확하고 막힌 부분 없음
2. **시각적 피드백**: 버튼 누름, 로딩, 진행률 표시
3. **터치 친화적**: 버튼 크기 최소 44x44pt
4. **오프라인 우선**: 모든 데이터 로컬 저장
5. **성능**: 이미지 최적화, 효율적인 렌더링
